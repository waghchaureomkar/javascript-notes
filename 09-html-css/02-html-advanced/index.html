1. HTML5 Canvas API

What is Canvas?

Canvas is used to draw graphics on the fly using JavaScript

Basic Canvas

Drawing Examples

Canvas Methods

// Context Methods
fillRect(x, y, width, height)    // Filled rectangle
strokeRect(x, y, width, height)  // Outlined rectangle
clearRect(x, y, width, height)   // Clear area

// Path Methods
beginPath()                       // Start new path
moveTo(x, y)                     // Move pen
lineTo(x, y)                     // Draw line
arc(x, y, r, start, end)         // Draw arc/circle
closePath()                      // Close path
fill()                           // Fill path
stroke()                         // Stroke path

// Styles
fillStyle = color               // Fill color
strokeStyle = color             // Stroke color
lineWidth = number              // Line width

// Text
fillText(text, x, y)           // Filled text
strokeText(text, x, y)         // Outlined text
font = "size family"           // Font style

2. SVG (Scalable Vector Graphics)

What is SVG?

SVG is XML-based vector graphics. Scalable, search-engine friendly, and accessible.

SVG Examples

Canvas vs SVG

Feature Canvas SVG
Type Raster (pixel-based) Vector (path-based)
Scalability Gets pixelated when scaled Scales without quality loss
Performance Better for many objects Slower with many elements
Accessibility Poor (just pixels) Good (can be read by screen readers)
SEO Not searchable Searchable (XML-based)
Event Handlers Must calculate positions Can attach to individual elements
Use Case Games, complex graphics Charts, icons, logos

localStorage vs sessionStorage

Feature localStorage sessionStorage
Persistence Permanent (until cleared) Only for session (tab close = clear)
Scope Shared across all tabs Specific to tab
Size Limit ~5-10MB ~5-10MB

Storage Methods

// Set item
localStorage.setItem('key', 'value');

// Get item
const value = localStorage.getItem('key');

// Remove item
localStorage.removeItem('key');

// Clear all
localStorage.clear();

// Get all keys
for (let i = 0; i < localStorage.length; i++) {
    const key = localStorage.key(i);
    console.log(key, localStorage.getItem(key));
}

4. Geolocation API

Get User Location

Geolocation Methods

// Get current position (one-time)
navigator.geolocation.getCurrentPosition(success, error, options);

// Watch position (continuous)
const watchID = navigator.geolocation.watchPosition(success, error, options);

// Stop watching
navigator.geolocation.clearWatch(watchID);

// Success callback
function success(position) {
    const lat = position.coords.latitude;
    const lon = position.coords.longitude;
    const accuracy = position.coords.accuracy;
}

// Error callback
function error(err) {
    console.log(err.code, err.message);
}

6. Web Accessibility (ARIA)

What is ARIA?

Accessible Rich Internet Applications - Makes web content accessible to people with disabilities

ARIA Attributes

role
Defines element purpose (button, navigation, main, etc.)
aria-label
Provides accessible name for element
aria-labelledby
References ID of element that labels current element
aria-describedby
References ID of element that describes current element
aria-hidden
Hides element from screen readers
aria-live
Announces dynamic content changes
aria-expanded
Indicates if element is expanded
aria-required
Marks required form fields

Examples

<!-- Custom button -->
<div role="button" tabindex="0" aria-label="Close dialog">X</div>

<!-- Navigation -->
<nav role="navigation" aria-label="Main navigation">...</nav>

<!-- Form with error -->
<input type="email" aria-required="true" aria-invalid="true"
       aria-describedby="email-error">
<span id="email-error">Please enter valid email</span>

<!-- Live region -->
<div aria-live="polite" aria-atomic="true">
    Content will be announced when updated
</div>

<!-- Expandable section -->
<button aria-expanded="false" aria-controls="content">
    Show More
</button>
<div id="content" hidden>...</div>

Accessibility Best Practices

  • ✅ Use semantic HTML (nav, main, article)
  • ✅ Provide alt text for images
  • ✅ Use proper heading hierarchy (h1 → h2 → h3)
  • ✅ Label all form inputs
  • ✅ Ensure keyboard navigation works
  • ✅ Use sufficient color contrast (4.5:1 for text)
  • ✅ Don't rely only on color to convey information
  • ✅ Provide skip navigation links
  • ✅ Use ARIA only when semantic HTML isn't enough
  • ✅ Test with screen readers

7. SEO Best Practices

Essential Meta Tags

<meta name="twitter:title" content="Page Title">
<meta name="twitter:description" content="Page Description">

SEO Best Practices

  1. Use Semantic HTML: header, nav, main, article, aside, footer
  2. Proper Heading Structure: One h1 per page, logical hierarchy
  3. Descriptive URLs: /blog/html-tutorial (not /page?id=123)
  4. Alt Text for Images: Descriptive and keyword-rich
  5. Internal Linking: Link to related pages on your site
  6. Mobile-Friendly: Responsive design, viewport meta tag
  7. Fast Loading: Optimize images, minify CSS/JS
  8. HTTPS: Secure connection
  9. Schema Markup: Structured data for rich snippets
  10. Sitemap: XML sitemap for search engines
